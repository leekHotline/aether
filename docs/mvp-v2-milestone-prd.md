# Aether 首页优化 PRD v2 — 轻量未来感（多元宇宙节点舞台）

## 1. 背景与目标

当前首页的“Write → World Reacts”过于单薄，无法体现“世界模型 IDE / 游戏”的核心体验。本版本目标是用**轻量未来感**的视觉与交互，将“规则被改写 → 世界响应”直观可感知。

**核心目标**
- 3 秒内让用户明白：这是“写一句话就能改变世界”的平台
- 形成一个可记忆的交互符号：**多元宇宙节点舞台**
- 保持留白与呼吸感，避免信息密度过高

**非目标**
- 不做真实世界模型推理
- 不引入复杂 3D 引擎（WebGL/Three）
- 不做账号/支付/协作系统

---

## 2. 设计方向

**风格关键词**
- 白底、柔和光晕、浮层卡片
- 轻、明、呼吸感
- 科幻但克制，避免机械厚重

**主交互**
- 多元宇宙节点舞台：4 个核心节点球体（Gravity / Time / Material / Force）
- 触发效果：连线发光 + Ripple 扩散 + 旁边卡片内容更新
- 默认聚焦 Gravity

---

## 3. 信息架构与页面结构

**Hero（主舞台）**
- 左侧：标题、副标题、主 CTA
- 右侧：多元宇宙节点舞台（SVG+DOM）
- 背景：Canvas 粒子雾 + 径向柔光晕

**节点语义卡片区（Hero 右下 / 下方）**
- 3–4 张浮层卡片，随节点切换内容
- 内容展示“规则变化 → 视觉结果”的短句

**价值阐释区（轻量）**
- 3 个短卡片说明：Write / Compile / World Reacts

**底部 CTA**
- 进入世界 / 观看 Demo

---

## 4. 核心组件清单

- `MultiverseStage`：节点舞台容器（SVG + DOM + Canvas 背景）
- `NodeSphere`：球体节点（渐变 + 内发光 + 外发光）
- `NodeLink`：节点连线（默认弱光，触发强光）
- `RippleBurst`：触发时的波纹扩散
- `OrbitCard`：浮层卡片（内容随节点切换）
- `GlowBackground`：背景光晕层
- `ParticleField`：Canvas 粒子雾层
- `HeroCTA`：主按钮组与辅助 CTA

---

## 5. 交互与动效规范

**入场动画**
- 标题与副标题：`fade-in + subtle up`
- 节点舞台：`scale-in + glow rise`
- 卡片：`stagger fade-in`

**节点交互**
- Hover 或 Click 节点：
- 连线亮度提升 2 倍
- 节点外发光增强
- Ripple 扩散（300–600ms）
- 右侧浮层卡片内容切换（fade + slide）

**呼吸感**
- 节点整体每 6–8s 轻微呼吸缩放
- 背景光晕随鼠标轻微漂移

**滚动反馈**
- Hero 向下滚动：舞台轻微缩小 0.95，透明度降低 10%

---

## 6. 节点语义内容规范

**节点列表**
- Gravity：`失重 / 漂浮 / 轨道失稳`
- Time：`慢动作 / 延迟 / 事件回放`
- Material：`材质变形 / 破碎 / 液化`
- Force：`冲击 / 推力 / 爆发`

**卡片结构**
- Title：节点名称
- Subtext：规则变化描述
- Effect：视觉结果短句

---

## 7. 技术实现约束

**技术栈**
- Next.js 14 + TailwindCSS
- Framer Motion（交互与卡片动画）
- GSAP（滚动与呼吸节奏）
- Canvas（背景粒子）
- SVG（节点与连线）

**性能约束**
- 60fps 优先
- Canvas 粒子在低性能设备降级或关闭
- `prefers-reduced-motion` 支持

---

## 8. 验收标准

**视觉**
- 留白 ≥ 60%
- 光晕层次清晰、不过曝
- 节点为球体感，非平面圆点

**交互**
- Hover/点击节点必触发 Ripple
- 连线发光同步变化
- 卡片内容 200ms 内完成切换

**性能**
- Hero 交互无明显卡顿
- Lighthouse Performance ≥ 90

## 10. 风险与应对

**风险 1：过度动效导致卡顿**
- 应对：Canvas 粒子降级，控制发光层数

**风险 2：视觉过空**
- 应对：增加卡片堆叠层次、微妙阴影与柔光

**风险 3：交互不直观**
- 应对：默认聚焦 Gravity，首次提示“点击节点”

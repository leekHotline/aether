# 创意点子工具 触发提示词: 使用创意点子工具：给我3个xxx的创意方案

## 目标
- 产出 3 个差异明显、可落地的创意方案
- 每个方案包含：核心隐喻、主交互、视觉语言、关键组件、动效要点、技术实现、风险与规避
- 给出 1 个推荐方案与明确下一步

## 输入处理
- 如果关键信息缺失，仅追问 **1 个**最重要问题（优先问目标页面/场景）。
- 如果用户明确“直接给方案”，跳过追问并使用默认假设：
  - 场景：Landing / Hero
  - 受众：创作者/开发者/投资人
  - 技术栈：Next.js + Tailwind + Framer Motion + GSAP（可选 Canvas/Three）
  - 目标：3 秒理解产品价值

## 输出格式（必须遵守）

**Summary**
- 一句话概述本次创意方向

**Concepts (3)**
每个概念按以下模板输出：
- 名称
- 一句话定位
- 核心隐喻
- 主交互
- 视觉语言（颜色/材质/空间层级/光感）
- 关键组件（3–6 个）
- 动效要点（3–5 条）
- 技术实现（贴合现有技术栈）
- 风险与规避

**Recommendation**
- 推荐哪一个 + 3 条理由

**Next Steps**
- 2–4 条可执行步骤（如：做低保真、打样动效、验收指标）

## 质量要求（去同质化）
- 三个概念必须在以下至少 3 点上明显不同：
  - 核心隐喻
  - 主交互
  - 视觉材质（玻璃/金属/织物/光尘等）
  - 布局结构（单核/双栏/环形/层叠）
- 避免常见“撞款”组件：纯 CountUp、空泛渐变背景、无语义的动效
- 每个概念必须包含至少 1 个独特“记忆点”

## 交付原则
- 不写代码，除非用户明确要求实现
- 不复制现成产品外观，只提炼风格
- 保持语言简洁、可落地
